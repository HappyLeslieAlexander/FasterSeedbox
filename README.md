# FasterSeedbox

Portable, auditable, POSIX-shell tuning for high-throughput BitTorrent/PT seedboxes.

- **Design**: small composable scripts, explicit side effects, idempotent writes.
- **Scope**: Linux (Debian/Ubuntu, RHEL-family, Alpine) and FreeBSD.
- **Interface**: one script per platform, plus `--dry-run` for safe inspection.

---

## 1) Philosophy

FasterSeedbox follows classic UNIX/POSIX principles:

1. **Do one thing well**: each script configures kernel/network/resource knobs for one OS family.
2. **Text as interface**: all state is persisted in plain-text drop-ins (`sysctl.d`, `limits.d`, init/service units).
3. **Composability**: runtime tuning is isolated in a helper script invoked by init/system manager.
4. **Idempotency**: reruns converge to the same target state.
5. **Auditability**: `--dry-run` previews file writes and operational steps before mutation.
6. **Fail-loud on critical paths**: required operations surface as explicit errors.

---

## 2) Supported Platforms

| Platform | Script | Init system |
|---|---|---|
| Debian / Ubuntu | `debian.sh` | systemd |
| RHEL / Rocky / Alma / CentOS | `rhel.sh` | systemd |
| Alpine Linux | `alpine.sh` | OpenRC |
| FreeBSD | `freebsd.sh` | rc.d |

> Use the script that matches your host OS. Do not cross-run scripts across families.

---

## 3) What It Changes

### Persistent configuration

- `/etc/sysctl.d/99-seedbox.conf` (or FreeBSD-equivalent sysctl include flow)
- `/etc/security/limits.d/99-seedbox.conf` (Linux PAM limits)
- system manager defaults where available (e.g., `DefaultLimitNOFILE` drop-ins)
- boot-time service definitions to re-apply runtime-only tuning

### Runtime tuning (re-applied at boot)

- NIC tx queue length, ring buffer sizing (when tooling/kernel support exists)
- route initial window tuning (`initcwnd` / `initrwnd` where supported)
- queue/scheduler selections based on block device class
- module load attempts for FQ/BBR where applicable

### Resource limits

- `nofile` targets are configured persistently for new sessions/services.
- helper scripts also attempt runtime `ulimit -n` raise as best-effort fallback.

---

## 4) Safety Model

- **Root required**: scripts validate effective UID.
- **Atomic writes**: temp file + move semantics for managed files.
- **Backups**: existing managed files are backed up with timestamp suffixes.
- **Dry-run first**: no system mutation when `--dry-run` is enabled.
- **Graceful degradation**: optional capabilities (e.g., ethtool features) may warn and continue.

## 5) Usage

### 5.1 Audit (recommended)

```sh
sudo ./debian.sh --dry-run
# or: ./rhel.sh --dry-run | ./alpine.sh --dry-run | ./freebsd.sh --dry-run
```

### 5.2 Apply

```sh
sudo ./debian.sh
```

### 5.3 Verbose diagnostics

Linux scripts support verbose command output:

```sh
sudo ./debian.sh --verbose
sudo ./rhel.sh --verbose
sudo ./alpine.sh --verbose
```

---

## 6) Verification

Run checks relevant to your platform.

### Linux

```sh
sysctl net.ipv4.tcp_congestion_control
sysctl net.core.default_qdisc
ulimit -n
```

### FreeBSD

```sh
sysctl net.inet.tcp.functions_default
sysctl net.inet.tcp.fastopen
ulimit -n
```

Also verify service presence/status through your init system (`systemctl`, `rc-update`/`rc-service`, or `service`).

---

## 7) Rollback

The scripts print rollback hints at completion. In general:

1. Disable/remove boot-time tuning service.
2. Remove managed drop-ins generated by FasterSeedbox.
3. Reload sysctl / daemon manager.
4. Reboot if you need a clean kernel-runtime reset.

Always review backup files (`*.bak-<timestamp>`) before permanent cleanup.

---

## 8) Operational Notes

- **PAM limits** apply to **new login sessions**, not retroactively to existing shells.
- **systemd DefaultLimit** affects services started by systemd after reload/restart.
- Runtime `ulimit` in helper scripts affects the helper process context and descendants.
- Containerized environments may restrict sysctl/module/network operations by design.
